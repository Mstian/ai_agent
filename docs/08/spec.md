# MewCode 第八阶段 · 项目记忆系统 Spec

## 背景

目前 MewCode 每次新会话启动都是「失忆」状态——不记得上次对话做了什么、不记得用户的编码偏好、不记得项目特有的技术约束。所有上下文全靠用户每次重新输入，效率低且体验差。

第七阶段已实现上下文压缩（工具结果存盘 + LLM 摘要），但它解决的是**单次会话内的 token 管理**，不跨会话。需要一个项目记忆系统来实现**跨会话的知识持久化**，让 Agent 从「每次失忆」变成「越用越懂你」。

核心思路：靠三套机制（项目指令文件、会话存档、自动笔记）实现工作记忆和长期记忆的分层管理。

## 目标

- **项目指令文件**：新会话启动时自动加载项目技术栈、代码规范、注意事项，无需用户每次重复说明
- **会话存档**：每次对话完整记录为 JSONL 文件，崩溃不丢数据，支持中断后恢复
- **自动笔记**：从对话中自动提取用户偏好、项目知识等长期记忆，下一次会话自动注入
- **记忆索引**：控制注入上下文的大小，在信息量和 token 消耗之间取得平衡

本阶段不做向量数据库、RAG 检索、团队记忆同步，留给后续阶段或专门项目。

## 功能需求

### F1: 项目指令文件 — 多层加载与优先级

项目指令文件是手写 Markdown，记录技术栈、代码规范、注意事项等。启动时按三层搜索加载：

1. **项目根目录** — `<project_root>/CLAUDE.md` 或 `<project_root>/AGENTS.md`
2. **项目配置目录** — `<project_root>/.mewcode/` 下的指令文件
3. **用户全局目录** — `~/.mewcode/` 下的指令文件

**优先级规则：** 项目级高于用户级。高优先级内容排前面，让模型优先遵循。加载顺序：先加载低优先级，再加载高优先级，高优先级内容插入到 prompt 前面。

**@include 引用机制：**
- 指令文件内支持 `@include <relative_path>` 语法引用其他文件
- 被引用文件的内容展开后插入到 @include 指令所在位置
- 嵌套深度上限 3 层
- 用 visited 集合（已访问文件的绝对路径集合）防环路引用
- 被引用路径必须是项目目录内的相对路径，拦截 `../` 跳出项目目录的路径

### F2: 会话存档 — JSONL 追加写

每次 Agent 对话完整记录到一个 JSONL 文件。

**文件位置：** `<project_root>/.mewcode/sessions/<session_id>.jsonl`

**会话 ID 格式：** `YYYYMMDD-HHMMSS-xxxx`（年月日-时分秒-4位随机字符），防同秒撞车。

**JSONL 格式：** 每行一条完整的 JSON 记录，代表一条 Message。开头追加 session 元信息行：
```json
{"type":"session_meta","session_id":"...","started_at":"...","cwd":"..."}
```

**追加写优势：**
- 追加快：`fs.appendFileSync` O(1) 追加
- 崩溃安全：最坏情况只丢最后一行（正在写的那行）
- 恢复友好：坏行可跳过，其余行完整
- 不单独维护 meta 文件：需要 ID/标题/消息数时直接扫 JSONL 算出，少一份要保持同步的状态

**写入时机：** 每条消息（user / assistant / tool）产生时立即追加一行，不做缓存。

### F3: 会话恢复

支持从上次中断的会话继续对话，处理以下异常情况：

**F3.1 坏行跳过：** JSONL 最后一行如果是残缺 JSON（写了一半崩溃），跳过该行，其余正常恢复。

**F3.2 工具调用未配对：** 恢复时如果最后一条消息是 assistant 的 tool_use，但后面没有对应的 tool_result，截断该 assistant 消息（移除未配对的 tool_use block）。

**F3.3 Token 超限处理：** 恢复的会话如果 token 数超过窗口限制，在注入前先执行一次压缩（复用第七阶段的 ConversationSummarizer），压缩后再注入。

**F3.4 时间跨度提醒：** 如果会话最后活跃时间距离当前超过 24 小时，恢复时自动插入一条 system 消息：
```
[会话恢复] 上次对话于 YYYY-MM-DD HH:MM，距今已 X 天 Y 小时。
以下是之前的对话摘要/记录，请基于此继续工作。
```

**F3.5 过期清理：** 启动时检查 sessions 目录，删除 30 天以上未修改的会话文件。清理时机为启动时异步执行，不阻塞主流程。

### F4: 自动笔记 — 四分类记忆

每轮 Agent Loop 自然停下后（模型最终回复无工具调用时），异步调 LLM 从对话中提取长期记忆。

**四分类：**
1. **用户偏好（user_preference）** — 编码风格、工具偏好、交互习惯
2. **纠正反馈（correction）** — 用户指出的错误和纠正方式
3. **项目知识（project_knowledge）** — 项目架构、技术决策、关键文件位置
4. **参考资料（reference）** — 外部文档链接、API 用法、配置说明

**存储格式：** 每条笔记一个带 frontmatter 的 Markdown 文件。

**存储位置：**
- 项目级笔记：`<project_root>/.mewcode/memory/`
- 用户级笔记：`~/.mewcode/memory/`

**更新机制：**
- 触发时机：Agent Loop 自然结束（模型最终回复无 tool_use block）后异步执行
- 更新方式：把对话摘要 + 现有记忆索引发给 LLM，由 LLM 判断新增/更新/合并/删除
- 去重：完全交给 LLM 判断——如果新信息和已有笔记重复或相似，LLM 决定是更新还是跳过
- 异步执行：不阻塞用户继续输入，更新过程静默后台完成

### F5: 记忆索引 — 请求前注入

在每轮 LLM 调用前，将记忆索引注入到上下文，相当于 Agent 已经「读过」长期记忆。

**索引文件位置：**
- 项目级：`<project_root>/.mewcode/memory/MEMORY.md`
- 用户级：`~/.mewcode/memory/MEMORY.md`

**索引内容：** 每条记忆一行链接 + 简要描述：
```markdown
- [用户偏好：使用中文回复](user_preference_zh.md) — 所有回复使用中文
- [项目知识：数据库连接配置](project_db_config.md) — PostgreSQL 连接串格式
```

**大小限制：** 索引文件控制在 200 行 / 25KB 以内（约 2-3K tokens）。超限时按最近更新时间排序，只保留最新的 200 行。

**注入方式：** 启动时加载索引内容，注入到 system prompt 的 `long_term_memory` 模块（已有模块，目前为空）。注入后模型首轮就能看到长期记忆。

**注入时机：** 会话启动时加载一次，后续如果自动笔记有更新，在下一轮 LLM 调用前刷新索引内容。

## 非功能需求

### N1: 崩溃安全
- JSONL 追加写保证最坏只丢最后一行
- 会话恢复时坏行可跳过，不影响其余数据
- 自动笔记异步写入，不阻塞主流程

### N2: 性能
- 项目指令文件加载 < 50ms（纯文件读取 + @include 展开）
- 会话 JSONL 追加写 < 1ms（单行追加）
- 记忆索引加载 < 10ms（读小文件）
- 自动笔记 LLM 调用异步非阻塞，不增加交互等待时间
- 过期会话清理异步执行，不阻塞启动

### N3: 可维护性
- 会话文件每行一条 JSON，可用 `head`/`tail`/`grep` 等标准工具直接查看
- 记忆文件为 Markdown frontmatter 格式，可用任意编辑器修改
- 项目指令文件为纯 Markdown，与 CLAUDE.md 兼容

## 不做的事

- ❌ 向量数据库（embedding + 向量检索）
- ❌ RAG 检索增强生成
- ❌ 团队记忆同步（多用户共享记忆）
- ❌ 记忆版本管理和回滚
- ❌ 会话搜索/全文检索
- ❌ 记忆导出/导入工具
- ❌ 项目指令文件的自动生成（纯手写）
- ❌ 会话文件压缩/归档（仅过期删除）

## 验收标准

- AC1: 项目根有 CLAUDE.md 时，启动后 system prompt 包含其内容，且排在用户级指令前面
- AC2: @include 引用另一个文件，被引用内容正确展开到引用位置
- AC3: @include 嵌套超过 3 层时，第 4 层被忽略并打印警告
- AC4: @include 引用 `../../../etc/passwd` 等跳出项目的路径时被拦截
- AC5: 对话开始后，每条消息实时追加到 JSONL 文件，kill -9 后最后一行可能残缺但其余行完整
- AC6: 启动时列出可恢复的会话（显示 ID、时间、消息数），用户选择后恢复
- AC7: 恢复一个有坏行的 JSONL 文件，坏行被跳过，其余正常加载
- AC8: 恢复的会话中最后一条 assistant 的 tool_use 无对应 tool_result 时，该 block 被移除
- AC9: 恢复一个 30 天前的会话，自动插入时间跨度提醒
- AC10: 启动时 30 天以上的会话文件被自动清理
- AC11: Agent 完成一轮对话后，自动笔记异步生成/更新，不阻塞用户继续输入
- AC12: 下一次新会话启动时，system prompt 包含上次提取的记忆索引
- AC13: 记忆索引文件不超过 200 行
- AC14: 所有现有测试继续通过，类型检查零错误
