# MewCode 第九阶段 · 命令系统 Spec

## 背景

目前 MewCode 的斜杠命令（`/plan`、`/exit`、`/compress` 等）全部硬编码在 `main.ts` 的 `processInput` 函数里，用 if/else 链匹配。随着命令增多，这个函数会越来越臃肿，且命令行为与 Agent 循环、界面渲染紧密耦合。

需要一个命令注册和分发机制：用户输入斜杠开头的命令时绕过 Agent 直接执行本地操作，响应快、省 Token、行为确定。

## 目标

- 建立命令注册中心，统一管理命令元数据和处理器
- 实现斜杠命令解析器，识别命令名和参数
- 用分发器在输入入口分流：命令走本地，非命令送 Agent
- 支持别名和 Tab 补全
- 抽一层界面控制接口，命令实现不绑定具体渲染框架
- 将现有 8 个硬编码命令迁移到新系统

## 功能需求

### F1: 命令注册中心

- 每条命令登记：名称、别名（可选）、简短描述、用法示例、命令类型、可选参数提示、是否隐藏、处理函数
- 注册时检测别名冲突（包括名称与别名冲突、别名与别名冲突），冲突直接抛错
- 启动阶段完成所有注册，运行时不变更
- 支持查询：按名称查找、列出所有（排除隐藏）

### F2: 命令解析器

- 识别斜杠 `/` 前缀，非斜杠开头直接返回 null
- 第一个空格前为命令名，之后为参数字符串
- 命令名转小写做到大小写不敏感
- 空输入早返回
- 未命中命令时返回 `/help` 引导信息

### F3: 命令类型三分法

- **local**（纯本地）：不涉及界面、不涉及对话，如 `/exit`
- **ui**（影响界面）：改变界面状态，如 `/clear` 清屏、`/plan` 切换模式
- **prompt**（提示词注入）：将预设提示词送进对话交给 AI，如 `/review` 注入代码审查 prompt

### F4: 界面控制接口

- 抽象一个 `UIContext` 接口，命令通过它操作界面而不绑定具体渲染框架
- 接口包含：显示消息、发送用户消息到 Agent、切换 Agent 模式、查询 token 用量、刷新状态栏
- 状态栏可联动显示模式标记（如 `[DEFAULT]` / `[PLAN]`）

### F5: 输入分流器

- 在用户输入入口（readline on line）处加分流逻辑
- 斜杠开头 → 解析命令 → 本地分发执行
- 非斜杠开头 → 原样送 Agent
- 命令执行返回结果后直接显示，不走 Agent 循环

### F6: 别名和 Tab 补全

- 命令注册时支持多个别名
- Tab 补全：单匹配直接补全命令名，多匹配弹菜单让用户选
- 隐藏命令不参与补全
- 补全只补命令名，不补参数

### F7: 十个内置命令

| 命令 | 类型 | 描述 |
|------|------|------|
| `/help` | local | 列出所有可见命令及用法 |
| `/compact` | ui | 触发对话压缩 |
| `/clear` | ui | 清空对话历史 + 清屏 |
| `/plan` | ui | 切换为规划模式（只读工具） |
| `/do` | ui | 切换为执行模式（全工具） |
| `/session` | local | 显示当前会话信息和存档状态 |
| `/memory` | local | 显示长期记忆列表和状态 |
| `/permission` | ui | 查看或切换权限模式 |
| `/status` | local | 显示 Token 用量、缓存命中、模式等运行状态 |
| `/review` | prompt | 注入代码审查 prompt，让 AI 审查当前代码变更 |

## 非功能需求

- N1: 命令解析 O(1)（HashMap 查找，不走 if/else 链）
- N2: 注册阶段别名冲突检查 O(n²)，但 n 很小（~10-20 条命令），启动时一次性开销
- N3: 命令处理器内不直接写 `process.stdout.write`，统一走 UIContext 接口

## 不做的事

- ❌ 用户自定义命令（留给 Skill 系统）
- ❌ 动态生成提示词模板
- ❌ 命令级权限控制
- ❌ 命令历史搜索（如 Ctrl+R）
- ❌ 参数自动补全

## 验收标准

- AC1: 输入 `/plan` 执行本地分发，不触发 Agent 循环
- AC2: 输入 `/do` 恢复执行模式，状态栏显示模式变化
- AC3: 输入 `/help` 列出全部可见命令，隐藏命令不出现
- AC4: 注册时别名冲突抛错，包含冲突命令名和别名
- AC5: `/PLAN`、`/Plan`、`/plan` 三种写法都能正确匹配
- AC6: Tab 补全 `/com` → 唯一匹配 `/compact` 时自动补全；`/p` → `/plan` `/permission` 等多个时弹菜单
- AC7: 所有现有测试继续通过，类型检查零错误
