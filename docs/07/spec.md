# MewCode 第七阶段 · 上下文压缩 Spec

## 背景

Agent 长时间工作会累积大量对话历史，尤其是工具调用结果（读文件、grep 搜索结果等），Token 消耗极快。DeepSeek 128K 窗口看似够用，但几轮带大文件读操作的对话就能填满，然后 API 报错，Agent 瘫掉。

第七阶段给 MewCode 装上两层压缩策略：轻量预防（工具结果自动存盘）加重量兜底（LLM 摘要），确保对话再长也不会溢出。

## 目标

- 两层压缩：第一层自动存盘大工具结果，第二层 LLM 摘要对话历史
- 用户消息原文保留，不被改写
- 压缩后补边界提示，防止模型脑补细节
- 手动触发 + 自动触发，三次连续失败熔断

## 功能需求

### F1: 第一层——工具结果存盘（轻量预防）

每次 LLM 调用前检查消息列表中的工具结果：

- **单条阈值**：单条 tool_result 内容超过 3000 字符 → 存盘到 `.mewcode/tool_results/<timestamp>_<tool_name>.txt`
- **合计阈值**：单条 assistant 消息后面的所有 tool_result 合计超过 8000 字符 → 挑最大的依次存盘直到合计低于阈值
- 存盘后在对话中替换为预览 + 文件路径：
  ```
  [工具结果已存盘: .mewcode/tool_results/20260617_143021_read_file.txt]
  预览: <前 200 字符>...
  完整内容请用 read_file 读取
  ```
- 用户消息和 assistant 文本回复不存盘，原文保留

### F2: 第二层——LLM 结构化摘要（重量兜底）

当整体对话历史估算 token 数超过 87K（100K 窗口 - 13K 安全余量）时触发。

**摘要范围**：从最早的消息开始摘要，保留尾部约 10K token（至少保留最后 5 条消息）的原文。

**摘要 Prompt 要求**：
- 必须禁止模型调用任何工具（tools 传空数组或 undefined）
- 要求模型先写分析草稿（thinking），再写正式摘要
- 草稿用完就丢，不进入最终摘要

**摘要结构（固定部分）**：
1. 任务目标 — 用户最初要做什么
2. 已完成工作 — 已执行的关键操作和结果
3. 关键发现 — 重要的文件内容、搜索结果、错误信息
4. 当前状态 — 正在做什么，下一步计划
5. 待解决问题 — 还没解决的事项

**摘要注入**：摘要作为 system 消息插入到对话历史中，替代被摘要掉的消息。被替换的消息从数组中移除。

### F3: 边界消息

压缩后自动插入一条 system 消息：
```
[上下文已压缩] 早期对话已摘要。如需之前工具结果的完整内容，请用 read_file 重新读取存盘文件。不要根据摘要脑补具体代码或文件内容。
```

### F4: 触发时机

- **自动触发**：每次 API 调用前先跑轻量预防 → 再跑重量兜底检查。自动触发留 13K 安全余量
- **手动触发**：用户输入 `/compress` 命令。手动触发余量收窄到 3K（激进压缩，因为用户明确想压）
- 轻量预防每次都跑（成本低），重量兜底只在超阈值时跑

### F5: Token 估算

不使用精确 tokenizer，用近似估算：
- 锚定上一次 API 返回的 `usage.input_tokens` 作为"上次 LLM 调用时的确切 token 数"
- 上次调用后新增的消息，按 `字符数 × 0.4` 估算为 token 增量（中文约 0.5，英文约 0.25，取 0.4 折中）
- 总估算 = 上次确切数 + 增量估算

### F6: 熔断机制

- 连续 3 次摘要调用失败（LLM 返回错误、超时等）→ 熔断，跳过后续摘要检查
- 熔断后仍然可以做轻量预防（工具结果存盘），只是不做重量摘要
- 熔断状态在 `/clear` 时重置

### F7: 手动触发命令

- `/compress` — 立即触发压缩（跳过自动阈值检查，使用 3K 余量）
- 手动触发不管当前 token 是否超阈值，强制执行

## 非功能需求

### N1: 对 Agent 透明
- 压缩在 Agent.run() 的每轮开头自动执行，Agent 不感知
- 摘要 LLM 调用不计入 Agent 循环次数

### N2: 性能
- 轻量预防 O(n) 遍历消息，单次 < 10ms
- 摘要 LLM 调用约 2-5 秒，不影响 Agent 后续执行

## 不做的事

- ❌ 精确 tokenizer（用近似估算）
- ❌ 摘要策略机器学习优化
- ❌ 工具结果存盘的自动清理（退出时可清理，但不是自动的）
- ❌ 分级压缩（如"轻度/中度/深度"）
- ❌ 压缩后自动恢复（摘要不可逆）

## 验收标准

- AC1: 单条工具结果超 3000 字符时，自动存盘并替换为预览
- AC2: 工具结果合计超 8000 字符时，挑大的存盘直到低于阈值
- AC3: 对话 token 超 87K 时，自动触发 LLM 摘要
- AC4: 摘要内容包含任务目标、已完成工作、关键发现等结构化部分
- AC5: 压缩后注入边界消息，提示模型不要脑补
- AC6: 用户输入 `/compress` 手动触压压缩
- AC7: 连续 3 次摘要失败后熔断，不再尝试
- AC8: 用户消息在压缩中保持原文不变
- AC9: 所有现有测试继续通过，类型检查零错误
